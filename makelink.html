Make Your Own Link

Create your own unblocker links using free DNS or deployment services

Table of Contents

  • Deploying via FreeDNS (free subdomain)
  • Deploying via Fastly
  • Deploying via Vercel (needs GitHub)
  • Deploying via Render (needs GitHub)

Making Links via FreeDNS

1

Go to freedns.afraid.org. If blocked, try on another device (you can't put this site through an unblocker).

2

Click "Sign Up for Free". If you already have an account, skip to step 5.

3

Fill out all the info. For name, use anything. For email, use a temp email provider like TempMail or SmailPro so you can make more links in the future.

4

Press sign-up and complete the email activation. If it errors out, just continue to the next step.

5

Go to the Shared Domain Registry. Alternatively: Subdomains > Add subdomain > Domain dropdown > "many, many more" > Shared Domain Registry.

6

Click a domain you'd like (e.g. chickenkiller.com). Pick one with low popularity - it's probably unblocked. Make sure it's public.

7

Make sure "A" record type is selected at the top (should be default).

8

Enter any subdomain name (yourName.chosenDomain.com).

9

For the IP destination, use one from the IP Table below. Choose the unblocker you want.

10

Press "Save", complete the captcha, and wait a few minutes for your site to register!

Making Links via Fastly

1

Go to fastly.com and sign up for free. Use any name/company info and a temp email.

2

Verify your account via email and log in.

3

Go to manage.fastly.com/home. Click CDN on the sidebar, then "Create service".

4

Service name: anything. Domain: yourname.global.ssl.fastly.net

5

For the Origin (host), put an unblocker URL. Don't include a path. Examples:

  • Astroid: https://astroid.gg/
  • Rammerhead: https://browser.rammerhead.org/
  • CanLite: https://canlite.org/
6

Click Activate. Then click "Edit Configuration" > "Clone Version 1 to edit".

7

Go to Settings, scroll to Fallback TTL and set it to 0 seconds. Press Activate.

8

Wait 30 seconds and test your link! If you get "503 Backend unavailable", the origin host doesn't work - try another one.

Troubleshooting: If it doesn't work, edit config > clone version > Settings > set Fallback TTL to 0, create a cache setting with TTL=0 and Stale TTL=0, activate, then purge all from service summary.

Making Links via Vercel

You need a GitHub account for this method.

1

Use this URL format: https://vercel.com/new/clone?repositoryurl=REPO_URL

Replace REPO_URL with an unblocker repo. Examples:

  • Interstellar: https://github.com/InterstellarNetwork/Interstellar
  • Doge: https://github.com/dogenetwork/v4
  • Utopia: https://github.com/UtopiaUnblocker/Utopia
  • Abyss: https://github.com/AbyssServices/Abyss-Web
  • Astroid: https://github.com/VyperGroup/astroidv3
  • Shuttle: https://github.com/shuttlenetwork/shuttle
  • Snorlax: https://github.com/Snorlaxs-Cave/Snorlaxs-Cave-v2
  • Artclass: https://github.com/art-class/v4
2

Log in to Vercel with GitHub (sign up if needed).

3

Enter any repository name and click "Create".

4

After a few moments, your link will be: repoName.vercel.app

Making Links via Render

You need a GitHub account for this method.

1

Go to render.com and sign up with GitHub.

2

Fork an unblocker repository on GitHub first (see Vercel section for repo links).

3

On Render, click "New" > "Web Service" and connect your forked repo.

4

Set the build and start commands (usually npm install and npm start). Choose the free plan and click Deploy.

5

Your link will be: repoName.onrender.com
